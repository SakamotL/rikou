# 日课 ·「账」Tab 结构设计方案（调研 + 深度思考 + 验证）

> 目的：重新设计「账」标签页的结构。先调研市面记账软件大结构，再拆解钱迹结构，深度思考后给出最终可落地的结构树。**本文只定结构，不改代码**，确认后再实现。

---

## 一、市面记账软件的"大结构"（共通范式）

无论随手记、鲨鱼记账、钱迹、MoneyWise，个人记账 App 的底层结构高度一致，可抽象为 5 层：

```
1. 账户/资产层   —— 钱从哪来、存在哪（现金/银行卡/信用卡/投资/负债）
2. 流水/交易层   —— 每一笔钱的进出（收支/转账/还款/债务…）
3. 分类层        —— 钱花在哪、从哪来（一级/二级分类 + 图标）
4. 预算/约束层   —— 给未来花钱设上限（总预算/分类预算）
5. 报表/洞察层   —— 回头看（饼图/趋势/资产曲线/结余）
```

两种记账范式：
- **单式记账（流水账）**：每笔只记"金额 + 分类 + 账户"，余额靠"账户加减"维护。个人记账 99% 用这个，简单直观。
- **复式记账（Beancount / 会计）**：每笔同时记"借/贷"两个科目，自动配平。适合生意账、公司财务，个人用过度复杂。

**结论**：日课是单人本地生活工具，用**单式记账 + 账户余额自动加减**即可，不引入复式。

## 二、钱迹的结构（官方文档拆解）

钱迹定位"三无"（无广告/无开屏/无理财），极简专注个人记账。其结构：

**设计理念**
- 打开即记一笔；数据本地/云端同步；资产管理不是会员专属。

**记账类型（交易枚举）**
- 收支、转账还款、周期分期、债务管理、报销管理、退款、手续费、优惠券

**账户/资产体系（四大类）**
- 资金账户：现金、银行卡、支付宝、微信、QQ钱包、京东金融、其它
- 信用卡账户：信用卡、花呗、京东白条、其它（欠款填**负数**）
- 充值账户：话费、公交卡、饭卡、会员卡、押金、其它充值卡
- 投资理财：股票、基金、其它理财（仅记录，不算收益）

**资产负债表（简化版）**
- 资产 + 负债 两部分；净资产 = 资产 − 负债
- 关键洞见：当月**结余**（收入−支出）≠ 净资产**增减**（报销、不计入账户会造成偏差）
- 债务需填"结束日期"才能从对应月份资产负债表移除

**分析/预算**
- 年/月/自定义；饼图、条形图、折线图；多维度分类统计
- 月度/年度总预算 + 分类预算；日均预算动态更新
- 自定义：月起止、多币种、分类名称排序、收支颜色、是否计入收支/预算/总资产

**日课应"砍掉"的钱迹功能**（过度，不适合单人本地工具）
- 多账本 & 共享记账、多币种、云同步会员体系、理财推荐 → 全部不照搬

## 三、深度思考（5 个关键决策 + 验证）

### 决策 1：记账范式 → 单式
个人生活账不需要复式配平。单式 + 账户余额自动加减，心智负担最低，符合"极简"。
✅ 验证：钱迹、鲨鱼、随手记个人版全都是单式；复式仅会计/生意场景。

### 决策 2：账户用 `kind: asset | liability` 区分（沿用现有设计）
钱迹把信用卡欠款记为负、债务单列。我们用 `kind` 字段把"账户"分为资产/负债两类，与现有 `accounts` 数据模型完全一致——**无需改数据层，只改展示与录入**。
✅ 验证：与已有 `netWorth()` / `defaults()` 中 `kind` 字段对齐，改动是增量不是重写。

### 决策 3：资产/负债占比 = 双视角（本方案最核心创新）
用户明确引用《穷爸爸富爸爸》，要"负债与资产占比"。这里有个**深层矛盾**需要点破：

- **会计视角（钱迹式）**：资产 = 账户余额（信用卡欠款为负计入负债）；负债 = 债务 + 信用欠款；净资产 = 资产 − 负债。回答"我拥有什么"。
- **穷爸爸视角（现金流）**：资产 = 持续把钱**放进**口袋的（收租房产、分红、版税）；负债 = 持续把钱**抽走**的（房贷月供 > 租金、消费贷）。回答"钱往哪流"。

→ 同一套账户数据，加一个 `cashflow` 标签（月净流 +/−），占比条就能**切换两种视角**：
  - 默认会计视角（和钱迹一致，稳妥）
  - 一键切穷爸爸视角（看"你的钱多少在生钱 vs 在漏钱"）

✅ 验证：不增加新数据表，只在账户上加可选 `cashflow` 标签；占比条双模式，满足用户"想看占比"的真实诉求（不是看数字，是看**钱在为你工作还是为你打工**）。

### 决策 4：价格目录必须归位「账」（用户重点）
价格目录本质是"我买过什么、花了多少"的**资产购买成本台账**，服务于"资产盘点 + 比价 + 抗通胀"，与"财务中枢"主题完全一致。从混乱的"库（记）"拎出，作为「账」的一等公民。
✅ 验证：与现有 `priceProducts()` 同名聚合、`buildSpark()` 趋势线天然契合，代码已具备，只需迁移到「账」Tab。

### 决策 5：内部预留"账本"概念，UI 不暴露
用户是电商运营，未来可能想要"生意账"和"生活账"分开。数据模型给每笔加 `book`（默认"个人"），**UI 暂不暴露多账本**，为未来留口子而不增加当下复杂度。
✅ 验证：零成本预留，不影响当前极简体验。

## 四、最终打印：「账」Tab 结构树

```
账 (Money) —— 财务中枢
│
├── 顶部常驻：净资产仪表盘
│   ├── 净资产（大数字，可切换 会计视角 / 穷爸爸视角）
│   ├── 资产合计 ─── 负债合计（双数字）
│   ├── 资产-负债占比条（可视化，双视角切换）
│   └── 本月结余（收入 − 支出，并标注"≠ 净资产增减"）
│
├── A. 账户体系（钱迹式四大类 + kind 区分）
│   ├── 资金类 (kind=asset)：现金 / 银行卡 / 支付宝 / 微信 / 其它
│   ├── 信用类 (kind=liability)：信用卡 / 花呗 / 白条（欠款为负）
│   ├── 投资类 (kind=asset)：股票 / 基金 / 其它理财
│   ├── 充值类 (kind=asset)：饭卡 / 公交卡 / 会员卡
│   └── 债务（独立模块）：借出 / 借入，带结束日期
│       每条账户：名称 · 类型 · 余额 · 计入总资产 · 隐藏 · (cashflow标签)
│
├── B. 记账（单式，自动加减账户余额）
│   ├── 交易类型：支出 / 收入 / 转账 / 还款 / 借出 / 收回 / 报销 / 退款 / 分期 / 手续费
│   └── 每笔字段：账户 + 分类 + 金额 + 日期 + 备注 + (book=个人)
│
├── C. 分类体系
│   ├── 一级分类（支出 / 收入）
│   ├── 二级分类
│   └── 图标网格（可自定义名称 / 排序 / 颜色）
│
├── D. 预算
│   ├── 月度总预算
│   ├── 分类预算
│   └── 日均可用 / 超支预警
│
├── E. 报表与趋势
│   ├── 本月收支（饼图 / 条形）
│   ├── 资产曲线（净资产随时间）
│   ├── 分类占比
│   └── 时间筛选（年 / 月 / 自定义）
│
└── F. 价格目录（用户重点，从"库"拎出）
    ├── 购买记录：商品 / 价格 / 日期 / 渠道 / 备注
    ├── 同名聚合（priceProducts）
    ├── 涨跌徽章（再买比上次贵 / 便宜）
    └── 价格趋势 sparkline（buildSpark）
```

## 五、与现有代码的映射（说明这是"增量"不是"重写"）

| 结构模块 | 现有 app.js 对应 | 需补 |
|---|---|---|
| A 账户体系 | `accounts`(含 kind)、`netWorth()` | 信用/投资/充值分类 UI、债务模块、cashflow 标签 |

| D 预算 | `settings` 预留 | 总/分类预算 + 日均 |
| E 报表 | （待建） | 饼图/资产曲线/筛选 |

| 顶部仪表盘 | `netWorth()` | 双视角占比条 + 切换 |
